Almost all computing languages allow you to define your own functions.

Why Define Functions?

Defining functions helps you organize code and avoid repetition. You write the steps once and reuse them whenever needed.

Parts of a Function

A function typically has:

  • A name: How you refer to and call the function
  • Parameters: Input values the function accepts (optional)
  • A body: The code that runs when the function is called
  • A return value: The result the function produces (optional)

Defining a Function

While exact syntax varies by language, the general pattern is:

function function_name(parameter1, parameter2):
    // Function body (the steps to perform)
    result = do_something(parameter1, parameter2)
    return result

Calling a Custom Function

After defining a function, you call it with arguments (actual values), just like an intrinsic function:

value = function_name(argument1, argument2)

Parameters vs. Arguments

Parameters are the placeholders in the function definition. Arguments are the values you pass when calling the function:

// Define a function with parameters x and y
function add(x, y):
    return x + y

// Call the function with arguments 5 and 3
result = add(5, 3)

In this example, x and y are parameters, and 5 and 3 are arguments. Inside the function call, x receives the value 5 and y receives the value 3.

Return Values

A function can return a value, but it doesn't have to. If a function doesn't return anything, it may be called for its side effects (like printing output or modifying data in a file).

Why Functions Matter

Functions are important for several reasons:

  • Reusability: Write code once, use it many times
  • Organisation: Break complex problems into smaller, manageable pieces
  • Readability: Named functions make code self-documenting
  • Maintainability: Fix a bug in one place rather than throughout your code
  • Testing: Test individual functions independently
